Members: SSMM import + welcome invite + email audit log

Phase 1 of migrating off the legacy SSMM WordPress plugin. Delivers the
full "import now, invite on launch" pipeline and the audit infrastructure
that underpins all future outbound mail from the app.

New:
- `members:import-ssmm {path} [--dry-run]`
  Imports the SSMM CSV export as-is. Skips `unverified` rows (never
  confirmed an email), imports `active`/`pending`/`expired`/`inactive`
  without sending any email. Maps "Standard Membership" -> full-member,
  "Junior" -> junior, "Life Membership" -> life-member, and expands the
  plugin's single-string discipline value ("both" -> ["centrefire",
  "rimfire"]). Handles UTF-8 BOM on the header row (the plugin always
  emits one) and normalises junk phone country-code values to "+27".
  Idempotent: re-running updates users/members matched by email. Users
  are created with a random unguessable password; members claim their
  account via the welcome-invite setup link.
- `members:send-welcome [--status=active*] [--email=*] [--resend]
  [--limit=N] [--dry-run]`
  Sends the "Welcome to PPRC — claim your account" mail. Defaults to
  status=active so accidentally running it without flags can never
  invite expired/inactive ex-members. Idempotent via email_logs:
  skips anyone who already has a `sent` MemberWelcomeInvite unless
  --resend is passed. Failures are logged as status=failed rows.
- `App\Mail\MemberWelcomeInvite` mailable + HTML template.
  The setup URL is Fortify's existing `password.reset` route with a
  freshly minted reset token, so the recipient lands on the real
  password-reset screen — no custom "claim" flow.
- `email_logs` table + `App\Models\EmailLog` + `LogSentEmail` listener
  on `Illuminate\Mail\Events\MessageSent`, wired in AppServiceProvider.
  Every outbound email from the app (contact form, welcome invite,
  future bulk comms) now writes an audit row automatically with
  to/from/subject/status/message_id/sent_at + CC/BCC/Reply-To context.
  Indexed on (to_email, mailable_class, sent_at) to keep the welcome
  idempotency check fast.
- `MemberStatus::Inactive` enum case so SSMM `inactive` rows carry the
  right semantics instead of being shoehorned into `resigned`.

Verified parse of the real 263-row SSMM export:
  261 imported (84 active, 94 pending, 41 expired, 42 inactive)
  2 skipped (unverified)
  0 unknown statuses / types / missing emails.
